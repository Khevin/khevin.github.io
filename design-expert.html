01 — Commands

Four verbs.
A shared vocabulary.

design-expert exposes itself through four commands. Each one points at a different mode of design work — the same modes a senior designer moves through every day. Type any of them in Claude Code with a target.

Cmd · 01 discovery
/design-expert:plan

Plan the design first.

Discovery before pixels. Audience, register, brand voice, anti-references — captured as a brief the rest of the work reads. Stops the model from one-shotting a generic mock.

$ /design-expert:plan checkout flow for a calm clinical SaaS
Cmd · 02 execution
/design-expert:build

Build it on the bar.

From brief to real screens, real components, real type. Reads your tokens, respects your system, ships code — not a Figma mock the model can't render.

$ /design-expert:build pricing page from PLAN.md
Cmd · 03 voice
/design-expert:write

Write copy that holds.

Display lines, microcopy, error states, empty states — written in the voice the brief committed to, not in default-AI breeziness.

$ /design-expert:write hero for a calm clinical SaaS
Cmd · 04 judgment
/design-expert:review

Review with a real reviewer.

The work gets walked, scored against the foundations, and graded. The reviewer cites the principle behind every note — no vibes-based feedback, no "looks great, ship it."

$ /design-expert:review src/pages/pricing.tsx

All four commands read the same skill files. Edits to SKILL.md propagate everywhere.

02 — The Library

A small library
of skills.

design-expert is markdown all the way down. Every file is a discipline. Every discipline is editable. You can read it, fork it, override a section in PROJECT.md, or hand the whole library to a junior on day one.

03 — The Pantheon

The designers
the skill listens to.

A small canon, deliberately curated. Each name has a file in design-gods/ with their work, their rules, and the moments when their judgment overrides the skill's defaults. This is the bar.

04 — Install

2 minute install

design-expert ships as a Claude Code plugin under the khev-tools marketplace. Add the marketplace once, install the plugin, restart your session — the four commands appear in your / menu.

Step · 01

Add the khev‑tools marketplace.

From inside Claude Code, open the Customize tab and add a plugin marketplace. Or run the command directly:

~/your-project $ /plugin marketplace add khev/khev-tools
# registers the marketplace · contains: design-expert
✓ marketplace added · 1 plugin available
Step · 02

Install design‑expert.

Open the plugin browser and install the plugin. Restart Claude Code so the commands register.

~/your-project $ /plugin install design-expert@khev-tools
# SKILL.md loaded · 11 markdown files · 13 design-gods
✓ design-expert installed
✓ /design-expert:plan · :build · :write · :review
Step · 03 · First run

Now plan something. Then build it. 

The healthiest first session is a brief, then a build, then a review. Each command writes its output to disk, so you can read what the skill thinks before it ships code.

~/your-project $ /design-expert:plan a calm onboarding for SREs on call
# writes PLAN.md · audience, register, brand voice, anti-references

~/your-project $ /design-expert:build onboarding from PLAN.md
# reads SKILL.md + PLAN.md · ships components in src/onboarding/

~/your-project $ /design-expert:review src/onboarding/
✓ scored against 10-lens framework · 4 notes · 0 blockers
05 — FAQ

Six things you might be wondering.

Most of what a designer or engineer asks before installing. Short answers here — the long ones are in the markdown.

  1. Q · 01
    What it is

    Is design-expert just a system prompt?

    No. It's a plugin: a manifest, a skill file, four commands, a 12-file markdown library, and a per-designer reference set. The skill files are loaded by the commands at the right moments — the reviewer doesn't need typography.md; the writer doesn't need output-format.md. Every byte of context earns its place.

  2. Q · 02
    Cost

    What does it cost?

    Nothing. Open source. Fork the markdown, override what you disagree with, send pull requests. The skill improves when more designers argue with it.

  3. Q · 03
    Compatibility

    Does it work with my design system?

    Yes — that's the point. The build command reads your tokens, components, and existing screens before it generates anything. design-expert grows with your system rather than around it. Ship a PROJECT.md with your conventions and the skill respects them.

  4. Q · 04
    Compatibility

    Cursor? Codex? Other harnesses?

    Coming. The skill files are pure markdown and portable; the wrapper is Claude-Code-shaped today. A universal package follows once the four commands stabilize.

  5. Q · 05
    Future

    Will there be other khev-tools plugins?

    Yes. design-expert is the first; the marketplace is built to host more. Until then, this is the entire shelf.

  6. Q · 06
    Naming

    Why "design-expert" and not something fancier?

    Because it does what it says. The plugin is what you reach for when you want a designer in the loop and there isn't one in the room. The name is the contract.
